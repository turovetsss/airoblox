0 Теория: что такое скрипт?

📜 Представь себе пульт управления

Скрипт — это как пульт от телевизора. Сам телевизор (объект) умеет показывать картинку, но без пульта ты не можешь переключить канал или изменить громкость. Скрипт даёт объекту команды: «стань больше», «поменяй цвет», «двигайся».

Скрипты пишутся на языке Lua (читается «Лу́а»). Этот язык очень простой, и сегодня ты напишешь свой первый код!

📂 Три вида скриптов — как разные пульты

В Roblox Studio есть три типа скриптов. Они похожи на пульты от разных устройств:

Тип скрипта Для чего нужен Где «живёт»
Script (серверный) Управляет всей игрой: врагами, дверями, правилами. Работает на сервере Roblox. ServerScriptService, Workspace
LocalScript (клиентский) Управляет тем, что видит только один игрок: интерфейс, камера, звуки. StarterPlayerScripts, StarterGui
ModuleScript (модульный) Библиотека готовых команд, которую можно использовать в других скриптах. Любой контейнер
🎯 Запомни простое правило: Если скрипт должен работать для всех игроков — выбирай Script. Если для одного игрока — LocalScript.

🔧 Свойства объектов — как кнопки на пульте

У каждого объекта в Roblox (например, Part) есть свойства — это как кнопки на пульте: «сделай громче» (размер), «переключи канал» (цвет), «включи/выключи» (прозрачность).

Вот самые главные кнопки для Part:

Свойство (кнопка)Что делаетПример команды в скрипте
SizeРазмер (ширина, высота, глубина)part.Size = Vector3.new(5,3,5)
PositionПозиция в миреpart.Position = Vector3.new(0,5,0)
AnchoredЗакрепить на месте (true/false)part.Anchored = true
TransparencyПрозрачность (0 — видимый, 1 — невидимый)part.Transparency = 0.5
BrickColorЦвет из палитры Robloxpart.BrickColor = BrickColor.new("Bright red")
MaterialМатериал (дерево, металл, стекло…)part.Material = Enum.Material.Wood
💡 Волшебное слово: чтобы скрипт узнал, к какому объекту он прикреплён, используй script.Parent. Это как сказать: «Тот, кто меня держит».

0 Как работает скрипт в игре?

Представь, что ты нажимаешь на кнопку — и что-то происходит. Вот как это выглядит внутри:

  1. Событие: игрок нажал на кнопку.
  2. Скрипт получает сигнал: игра замечает событие и передаёт его скрипту.
  3. Скрипт выполняет команды: например, меняет цвет Part или открывает дверь.
  4. Изменения применяются в игре: ты видишь результат!
⚙️ Важно знать: Скрипты работают на сервере игры. Они управляют логикой и делают игру интерактивной и живой!

0 Без скрипта — стоит, со скриптом — действует!

Сравни: обычный куб без кода и куб, которым управляет скрипт.

Скрипт делает объекты живыми — они могут двигаться, меняться и взаимодействовать с игроком.

📘 Шпаргалка юного скриптера

Основные правила и команды, которые пригодятся в каждом скрипте:

📌 Переменные и объекты
local part = script.Parent — ссылка на объект, в котором лежит скрипт.
local part = workspace:WaitForChild("Имя") — поиск Part по имени.
🔧 Изменение свойств
part.Size = Vector3.new(5, 3, 5)
part.Position = Vector3.new(0, 5, 0)
part.Anchored = true (закрепить)
part.Transparency = 0.5 (0 — видимый, 1 — невидимый)
part.BrickColor = BrickColor.new("Bright red")
part.Material = Enum.Material.Wood
🕹️ События
part.Touched:Connect(function(hit) ... end) — при касании.
click.MouseClick:Connect(function() ... end) — при клике (нужен ClickDetector).
⏳ Задержка и вывод
wait(2) — пауза 2 секунды.
print("Привет!") — вывод в окно Output.
❓ Условия (if/else)
if part.BrickColor == BrickColor.Red() then
  ... else ...
end
🔄 Циклы
for i = 1, 5 do ... end — повтор 5 раз.
while true do ... end — бесконечный цикл (осторожно!).
🧩 Полезные функции
BrickColor.Random() — случайный цвет.
Instance.new("Part") — создать новую Part.
part.Parent = workspace — поместить объект в мир.
⚠️ Важные правила
📌 Всегда пиши local перед переменной.
📌 Комментарии начинай с --
📌 Проверяй, существует ли объект: if part then
📌 Не забывай закрывать функции и условия словом end

⭐ Сохрани эту шпаргалку — она пригодится во всех следующих занятиях!

0 Как создать скрипт — пошаговая инструкция с Бобиком

1 Открой Roblox Studio и создай новый проект (Baseplate).
2 В окне Explorer найди ServerScriptService. Нажми на него правой кнопкой мыши → Insert Object → Script.
3 Дважды кликни на скрипт — откроется редактор кода. Там уже будет строчка print("Hello world!").
4 Напиши свой код. Например:
local part = script.Parent
part.Size = Vector3.new(10, 5, 10)
part.BrickColor = BrickColor.new("Really red")
(Скрипт в ServerScriptService не имеет Parent-объекта, поэтому создай скрипт внутри Part)
5 Нажми F5 (Play), чтобы запустить игру. Скрипт выполнится, и ты увидишь результат!
6 Чтобы увидеть сообщения от print(), открой Output (меню Script → Output).

📦 Задание 1 — Настраиваем Part (пульт в действии)

Создай скрипт, который установит для Part параметры:

  • Размер: 5, 3, 5
  • Позиция: 0, 10, 0
  • Якорь (Anchored): true
  • Прозрачность: 0.3
Выполнено шагов: 0 из 3

  • Готовый скрипт:

    local part = script.Parent

    part.Size = Vector3.new(5, 3, 5)
    part.Position = Vector3.new(part.Position.X, 10, part.Position.Z)
    part.Anchored = true
    part.Transparency = 0.3
✅ Проверь себя: Part должен висеть в воздухе на высоте 10, быть полупрозрачным и не падать.

🔘 Задание 3 — Волшебная кнопка (ClickDetector)

Создай кнопку, которая при нажатии меняет цвет на случайный.

Выполнено шагов: 0 из 3

  • Готовый скрипт:

    local part = script.Parent
    local click = part:FindFirstChild("ClickDetector")

    function onClick()
      part.BrickColor = BrickColor.Random()
      print("Кнопка нажата!")
    end

    click.MouseClick:Connect(onClick)
🎉 Весело! Каждый клик — новый случайный цвет. Можно сделать кнопку-сюрприз.

🌟 Задание 4 — Твой уникальный объект (свободное творчество)

Придумай свою собственную Part и настрой её через скрипт. Используй минимум 5 разных свойств из таблицы выше.

Примеры идей:

  • Тонкая пластина 0.5, 10, 0.5 цвета «Teal» и материал Glass (стекло).
  • Огромный золотой шар (Material = Metal, BrickColor = “Gold”).
  • Невидимый куб (Transparency = 1), который стоит на голове персонажа.

Добавь ClickDetector и придумай, что будет происходить при клике (можешь менять цвет, размер или выводить сообщение).

🏆 Бонус для смелых: добавь в скрипт строчку part.Shape = Enum.PartType.Ball и преврати куб в шар!

🚦 Задание 5 — Мини-светофор

Создай три Part (красный, жёлтый, зелёный) и скрипт, который по очереди «зажигает» их, имитируя светофор.

Алгоритм:

  1. Создай три Part, назови их RedLight, YellowLight, GreenLight. Раскрась соответственно.
  2. В одном общем Script (например, в ServerScriptService) напиши код, который бесконечно переключает цвета.
  3. Используй wait(секунды) для пауз.
Выполнено шагов: 0 из 3

  • Пример скрипта:

    -- Получаем части по имени из Workspace
    local red = workspace:WaitForChild("RedLight")
    local yellow = workspace:WaitForChild("YellowLight")
    local green = workspace:WaitForChild("GreenLight")

    while true do
      red.BrickColor = BrickColor.Red()
      yellow.BrickColor = BrickColor.Gray()
      green.BrickColor = BrickColor.Gray()
      wait(3)
      red.BrickColor = BrickColor.Gray()
      yellow.BrickColor = BrickColor.Yellow()
      wait(1)
      red.BrickColor = BrickColor.Gray()
      yellow.BrickColor = BrickColor.Gray()
      green.BrickColor = BrickColor.Green()
      wait(3)
    end

    Не забудь, что все три Part должны быть Anchored = true!

🚗 Интересный факт: такой же принцип используется в настоящих светофорах — программа по очереди включает лампы.

🌀 Задание 6 — Телепортация по клику

Создай кнопку, которая при нажатии перемещает твоего персонажа в другую точку!

  • Создай две Part: одна будет кнопкой (назовём TeleportButton), вторая — точкой назначения (Destination).
  • В TeleportButton добавь ClickDetector.
  • Напиши скрипт, который при клике берёт персонажа игрока и перемещает его на позицию Destination.
  • Подсказка: используй player.Character и CFrame.
Выполнено шагов: 0 из 3

  • Готовый скрипт (в TeleportButton):

    local button = script.Parent
    local click = button:FindFirstChild("ClickDetector")
    local destination = workspace:WaitForChild("Destination")

    function onClick(player)
      local character = player.Character
      if character then
        character:MoveTo(destination.Position)
      end
    end

    click.MouseClick:Connect(onClick)

    Убедись, что обе Part закреплены (Anchored = true).

🌠 Попробуй улучшить: поставь несколько точек назначения и кнопок, чтобы создать систему быстрого перемещения.

🔢 Задание 7 — Счётчик нажатий

Создай кнопку, которая считает, сколько раз на неё нажали, и меняет цвет при каждом клике.

  • Создай Part и назови его CounterButton.
  • Добавь ClickDetector и Script.
  • В скрипте заведи переменную-счётчик (local count = 0).
  • При каждом клике увеличивай счётчик на 1, выводи его в Output и меняй цвет Part.
Выполнено шагов: 0 из 3

  • Готовый скрипт:

    local part = script.Parent
    local click = part:FindFirstChild("ClickDetector")
    local count = 0

    function onClick()
      count = count + 1
      print("Кликов: " .. count)
      part.BrickColor = BrickColor.Random()
    end

    click.MouseClick:Connect(onClick)
🎯 Усложни задачу: сделай так, чтобы после 10 нажатий кнопка становилась зелёной и выводила сообщение «Победа!».
